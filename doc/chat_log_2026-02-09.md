## Methodology correction: mounting bias reference

An important correction was made:

- **Mounting bias** must be estimated from **PX4 attitude vs platform pose** (top plate frame).
- Using **X‑Plane attitude** as the mounting-bias reference was rejected as conceptually invalid.
- The chosen reference available in logs is the **commanded platform pose** in `tick.csv`:
  - `cmd_roll_deg`, `cmd_pitch_deg`, `cmd_yaw_deg`

## Code changes introduced

### 1) `postprocess_run.py`

Added mounting-bias estimation and bias-corrected tracking metrics:

- Reads `scenario_phase` from `tick.csv`.
- Estimates mounting bias in a steady window:
  - Prefer `scenario_phase == "hold_0"` and `pid_enabled == 1` (when present), within the analysis window.
  - Computes (roll/pitch):
    - `bias = mean(wrap(px4 - cmd))`
- Emits:
  - `mounting_bias_deg` in the output metrics JSON.
  - Additional error series in `tracking_error_deg`:
    - `roll_px4_biascorr_minus_xp_deg`, `pitch_px4_biascorr_minus_xp_deg`
    - `roll_px4_biascorr_minus_cmd_deg`, `pitch_px4_biascorr_minus_cmd_deg`
- Pitch response plot updated to include:
  - `X-Plane pitch + mounting bias` (shifted by the estimated bias) for visual alignment.

### 2) `paper/Paper_draft.md`

Updated Section 6–7 and Conclusion to match the new method:

- States that **pre/post bias-correction** tracking errors are presented.
